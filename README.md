# E-Commerce Transaction Data Analysis for Business Insights Using Python

# Project Summary:-
This project analyzes 100,000 **e-commerce transaction records** to uncover insights related to **sales performance, customer behavior, city-wise revenue distribution, and payment method trends**.
The analysis was performed using Python and Pandas in Jupyter Notebook (VS Code), focusing on business-driven insights and measurable KPIs.

-----

# Dataset Description:-
File: "Transaction_Raw_Data.csv"

# Columns Name:
- transaction_id
- customer_id
- city
- amount
- status
- payment_method
- transaction_date
- high_value_flag (engineered column)

-----

# KPIs Calculated:-
- Total Revenue.
- City-wise Revenue.
- City-wise Average Transaction Amount.
- Payment Method Usage.
- Transaction Status Distribution.
- High-Value vs Low-Value Transactions (%).
- Data Quality Check. (Null Values)

-----

# Business Objectives:-
- Identify high-performing cities and revenue drivers.
- Understand customer payment preferences.
- Measure transaction success and failure rates.
- Detect high-value transactions contributing maximum revenue.
- Support business decision-making using data insights.

-----

# Key Analytical Insights:-
- A small percentage of high-value transactions contribute a significant portion of total revenue.
- Metro cities dominate revenue generation, indicating higher purchasing power.
- UPI and Credit Cards are the most preferred payment methods.
- Cancelled and failed transactions highlight areas for process and payment optimization.
- Average transaction value varies significantly across cities.
